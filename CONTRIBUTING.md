# Contributing

Thanks for considering a contribution to SuperDoc Editor.

## Development

```sh
npm install
npm run dev
```

Before opening a pull request, run:

```sh
npm run type-check
npm run build
```

## Pull Requests

- Keep changes focused and reviewable.
- Include screenshots or a short recording for visible editor behavior changes.
- Document new iframe messages or config variables in `README.md`.
- Do not commit secrets, private keys, `.env` files, `dist/`, or `node_modules/`.
